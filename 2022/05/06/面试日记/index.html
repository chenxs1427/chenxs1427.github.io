问：apk脱壳的原理是什么？

答：fridaDEXDump主要是结合frida框架，通过在内存中暴力搜索dex035，来找到已经加载的dex文件，对于一些没有文件头的dex文件，则通过匹配一些特征来找到。

问：beautifulsoup有多少种类型的解析器？

答：有html.parser（python的内置标准库）、xml、lxml（支持解析html和xml，速度快，需要安装c语言库）、html5lib（有最好的容错性，但解析速度慢）

问：对于asyncio的理解

答：使用 async await 关键字
把函数包装成 task 注册进 事件循环
当遇到io耗时的操作时，则自动切换到task列表中的其他任务

问：什么情况下会发生GIL锁

答：io密集时，GIL是python的全局解释器 ，同一进程中假如有多个线程运行，一个线程在运行的时候会霸占python解释器（加了一把锁即GIL），使该进程内的其他线程无法运行，等该线程运行完后其他线程才能运行。如果线程运行过程中遇到耗时操作，则解释器锁解开，使其他线程运行。所以在多线程中，线程的运行仍是有先后顺序的，并不是同时进行。
多进程中因为每个进程都能被系统分配资源，相当于每个进程有了一个python解释器，所以多进程可以实现多个进程的同时运行，缺点是进程系统资源开销大

问：twisted框架异步的机制。

答：Twisted 框架的程序基本上都是通过 reactor事件循环 驱动回调函数，完成业务逻辑。reactor 循环一般运行于主线程中，由 reactor.run() 函数启动， reactor.stop() 函数退出循环。

问：给定一条sql语句，问mysql执行的过程是什么样的？

当执行一条查询的SQl的时候大概发生了一下的步骤：

  1. 客户端发送查询语句给服务器。
  2. 服务器首先检查缓存中是否存在该查询，若存在，返回缓存中存在的结果。若是不存在就进行下一步。
  3. 服务器进行SQl的解析、语法检测和预处理，再由优化器生成对应的执行计划。
  4. Mysql的执行器根据优化器生成的执行计划执行，调用存储引擎的接口进行查询。
  5. 服务器将查询的结果返回客户端。

问：高匿名代理、匿名代理、透明代理的区别

  • 透明代理，对方服务器可以知道你使用了代理，也知道你的真实IP；
  • 匿名代理隐藏了你的真实IP，但是对方知道你是使用代理服务器访问他们的。
  • 高匿名代理隐藏了真实IP，同时访问对象也不知道你使用了代理，因此隐蔽度最高。

问：https的一个加密过程是什么样的？

  1. 客户端发起连接（TCP三次握手）
  2. 将【CA证书】返回给客户端。证书里含有【公钥】
  3. 客户端根据【内置根证书】验证CA证书的合法性，然后生成随机的【对称加密密钥】
  4. 密钥通过公钥加密，发给服务端
  5. 双方通过密钥加密来进行http通讯

问：索引为什么快？

答：索引其实就是拿 数据库表中一列或多列的值进行排序的一种数据结构（比如mongodb是b树，mysql是b+树），在这棵树上检索数据的时候可以使用比较快的查找算法,如常用的二分查找，另外索引占用的资源比较小，一般可以在内存中读取，减少了磁盘的io操作。

问：Redis 的持久化机制是什么？各自的优缺点？

Redis提供两种持久化机制 RDB 和 AOF 机制：

RDB（Redis DataBase）持久化方式：

是指用数据集快照 的方式半持久化模式记录 Redis 数据库的所有键值对，在某个时间点 将数据写入一个临时文件 ，持久化结束后，用这个临时文件替换上次持久化的文件，达到数据恢复。
RDB优点：
性能最大化，主进程负责处理命令，不会进行任何 IO 操作，使用单独子进程来进行持久化，保证了 Redis的高性能。
当初始数据量大时，比 AOF 的启动效率更高。
RDB缺点：
容易造成数据丢失，RDB 是间隔一段时间进行持久化 ，如果这期间redis发生故障……

AOF（Append-only file）持久化方式：

是指所有的命令行记录 以 Redis 命令请求协议的格式完全持久化存储保存为 aof文件 。
AOF优点：
数据安全，通过 append 模式写文件，即使中途服务器宕机，可以通过 redis-check-aof 工具解决数据一致性问题。
AOF缺点：
AOF 文件比 RDB 文件大，且恢复速度慢。

问：AST混淆还原的原理是什么？

答：主要是把混淆后的js代码解析成抽象语法树，这颗树上的每个节点都类似一个json对象，它就代表源码里面的一种结构，比如表达式、声明、方法、模块什么的，然后就可以根据混淆的手法，对不同类型的节点进行转换和改写，这样就实现了混淆代码的还原。

问：wireshark的抓包原理是什么，和fiddler有什么不同？

异同

  • Fiddler是在windows上运行的程序，专门用来捕获HTTP，HTTPS的。
  • wireshark能获取HTTP、HTTPS和一些其他的网络协议TCP、UDP，但是不能解密HTTPS

抓包原理

  • fiddler是使用中间人代理的方式，所有的http请求在达到服务器之前都会经过fiddler，所有的http响应都会在返回客户端之前流经fiddler。
  • wireshark是采用直接绑定本机网卡的方式，即直接抓取进出本机网卡的流量包

进程间的通信方式有哪些？

管道( pipe )

管道是一种半双工的通信方式，数据只能单向流动，而且只能在具有亲缘关系的进程间使用。进程的亲缘关系通常是指父子进程关系。

信号量( semophore )

信号量是一个计数器，可以用来控制多个进程对共享资源的访问。它常作为一种锁机制，防止某进程正在访问共享资源时，其他进程也访问该资源。因此，主要作为进程间以及同一进程内不同线程之间的同步手段。

消息队列( message queue )

消息队列是由消息的链表，存放在内核中并由消息队列标识符标识。消息队列克服了信号传递信息少、管道只能承载无格式字节流以及缓冲区大小受限等缺点。

共享内存( shared memory )

共享内存就是映射一段能被其他进程所访问的内存，这段共享内存由一个进程创建，但多个进程都可以访问。共享内存是最快的 IPC 方式，它是针对其他进程间通信方式运行效率低而专门设计的。它往往与其他通信机制，如信号两，配合使用，来实现进程间的同步和通信。
